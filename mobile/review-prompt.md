# 전체 코드베이스 리뷰 요청

## 프로젝트 개요

KBO(한국프로야구) 직관/집관 기록 다이어리 앱입니다. React Native (Expo) 모바일 앱 + FastAPI 백엔드 서버로 구성.

### 구성
| 영역 | 언어/프레임워크 | 규모 |
|------|----------------|------|
| **모바일 앱** (`mobile/`) | TypeScript, React Native (Expo Router), SQLite | ~81,000 lines / 130+ 파일 |
| **서버** (`server/data_api/main.py`) | Python FastAPI, JSON 파일 데이터 | ~710 lines |
| **공유** (`shared/`) | TypeScript 타입 정의, Zod 스키마, 상수, API 클라이언트 | ~1,500 lines |

### 주요 기능
- 오늘의 경기 스코어 실시간 조회 (경기전/중/종료)
- 경기 상세 (라인스코어, 투수 정보)
- 직관/집관 일기 기록 (점수, 감정, 사진, 비용)
- 사용자 통계 (승률, 연승, 상대전적, 홈/원정, 구장별, 요일별)
- 스티커 생성 (해시태그, 배경 선택, 클립보드/공유)
- 업적/배지 시스템 (50+ 배지)
- 승리 토템 시스템
- 구장 정보
- 갤러리 (사진 타임라인, 검색)
- 팀 순위 (2020-2025)
- 온보딩 프리페치 최적화 (8회 API → 1회)

## 리뷰 요청 사항

아래 관점에서 전체 코드베이스를 평가해주세요. 각 항목마다 **구체적인 파일 경로와 라인 번호**를 함께 지목해주세요. 전체를 다 볼 수 없다면 핵심 파일(모바일: `lib/`, `components/`, `app/`, 서버: `main.py`, 공유: `shared/`)을 우선 검토해주세요.

### 1. 보안 (Security)
- 서버 API 인증/인가 상태
- 사용자 입력 검증 (모바일 → 서버)
- SQLite 캐시 민감 데이터 저장 가능성
- 클립보드/공유 기능 정보 노출
- Zod 스키마와 실제 API 응답 불일치 가능성
- CORS, Rate limiting, 에러 핸들링

### 2. 확장성 (Scalability)
- JSON 파일 기반 데이터 저장의 한계
- 병목 가능 API 엔드포인트
- SQLite 캐시 전략 (TTL, 무효화, 동기화)
- 온보딩 consolidation 접근법 장단점
- module-level promise 중복 호출 방지 패턴 평가

### 3. 사용자 경험 (UX)
- 오프라인/네트워크 불량 상황 동작
- 로딩 상태 표시 (Skeleton, Spinner, Toast)
- 데이터 정합성 (날짜 포맷 불일치 → 잘못된 통계)
- 경기 상태 전환 (전→중→종료) UI 처리
- 낙관적 업데이트 vs 실제 API 결과 충돌 대응
- 온보딩과 재진입 사용자 체감 차이

### 4. 발전 가능성 (Extensibility)
- 글로벌 상태 관리 일관성 (Context, SQLite)
- 새 기능 추가 시 수정 범위가 큰 구조
- Zod 스키마 런타임 검증 활용도
- 서버 헬퍼 함수 재사용성
- Expo 라우터 구조 확장성
- 컴포넌트 분할 적절성

### 5. 예상치 못한 버그 — Edge Cases
아래 상황에서 발생 가능한 버그를 집중 찾아주세요:

**날짜 처리**
`2026.05.30` vs `2026-05-30` 혼용, 연월 경계, 서머타임, 윤년

**경기 데이터**
시범경기/포스트시즌/올스타전 game_type 누락, 더블헤더 gameIdx 0/1 기반 vs 1-based 혼동, 취소 경기 통계 포함 여부, 무승부 streak 처리, 라이브 경기 스티커 null 처리

**동시성**
API 응답 순서로 캐시 덮어쓰기, 중복 프리페치, 화면 전환 중 로딩

**데이터 없음**
신규 사용자 빈 기록, 비시즌/월요일 경기 없음, 특정 연도/팀 데이터 부재

**스티커/통계**
가상 레코드가 실제 통계와 섞이는 문제, `resolveIsWin()` null/0/-1 처리, `filterByYear()` 날짜 포맷 불일치 누락, 해시태그 길이 제한

**서버**
JSON 파일 파손/공백, 외부 수집 실패 fallback, 메모리 캐시 60초 TTL 만료 직후 thundering herd

### 6. 코드 품질 (Code Quality)
- `any` 우회, 타입 단언(`as`), `!` 연산자 사용
- 불필요한 재렌더링 (useMemo/useCallback)
- useEffect 의존성 배열 누락
- 에러 처리 범위와 복원
- 중복 코드, 책임 분리

### 7. 테스트
- 테스트 커버리지 현황 (`shared/__tests__/`)
- 테스트하기 어려운 구조
- 가장 먼저 추가해야 할 테스트
