# 페이지별 코치마크 검토 요청

## 배경

KBO 직관 기록 다이어리 앱(fullcount.kr)에 인라인 툴팁 방식의 코치마크를 도입 중입니다.
사용자가 각 탭/페이지에 **처음 접근할 때** 간단한 툴팁으로 핵심 기능을 안내하려고 합니다.

## 현재 구현된 코치마크 패턴

홈 탭에 적용한 예시가 `components/HomeCoachMark.tsx`에 있습니다. 특징:
- **Modal 아님** — 일반 View가 레이아웃 안에 인라인으로 위치
- 확인 버튼 없음 — 카드 탭하거나 콘텐츠를 스와이프하면 자연스럽게 사라짐
- `user_settings` 테이블에 `has_seen_{page}_coach` 플래그로 1회만 노출
- 플래그는 노출 전에 미리 저장 (도중 이탈 시에도 다시 안 뜸)

코드:
```tsx
// props: visible, onDismiss
// visible=false → null 반환 (렌더링 안 함)
// 카드 전체가 Pressable, onPress={onDismiss}
// onDismiss에서 플래그 저장 + show=false
```

## 요청 사항

각 페이지별로 아래 형식으로 답변해주세요:

### `파일명`

- **코치마크 필요?** (필요 / 불필요 / 선택)
- **대상 UI 요소**: 가리킬 구체적인 UI 요소
- **추천 문구**: 한 줄 (14자 이내 권장)
- **이유**: 왜 이 가이드가 필요한지 (이 기능이 눈에 띄지 않거나, 일반 사용자가 모를 만한 기능)

### 평가 기준

- **필요**: 일반 사용자가 발견하기 어렵거나, 앱의 핵심 가치를 전달하는 기능
- **불필요**: UI가 직관적이어서 설명 없이도 알 수 있는 기능
- **선택**: 있으면 좋지만 없어도 크게 문제되지 않는 기능

### 제약 조건

1. **최소주의**: 꼭 필요한 것만. 너무 많은 가이드는 오히려 방해.
2. **한 줄 원칙**: 문구는 짧을수록 좋음 (14자 이내 권장)
3. **액션 중심**: "이곳을 탭하세요"보다 "여기서 기록을 확인하세요"처럼 가치 중심
4. **홈 탭은 이미 구현됨**: 홈 탭은 "좌우로 스와이프하여 날짜를 이동하세요" 이미 적용 완료

## 출력 형식

```
### app/(tabs)/diary.tsx
상태: 필요
대상: 우측 하단 ➕ FAB 버튼
문구: "직관 기록은 여기서"
근거: FAB는 즉각적으로 용도를 알기 어려움. 첫 방문 시 일기 작성으로 이끌어야 함.
```
