9	## 배경
10	
11	모바일 앱(fullcount.kr)의 경기 상세 화면에서 사용자가 직관한 경기 결과를 인스타그램 스토리에 공유할 수 있는 동적 PNG 스티커 생성 기능.
12	사용자가 경기장에서 찍은 사진 없이도 앱 데이터(점수, 직관 통계, 연승)로 즉석 스티커를 만들어 클립보드에 복사해준다.
13	
14	## 데이터 흐름
15	
16	```
17	game/[id].tsx [스티커 만들기] 버튼 (isFinished && gs)
18	  → StickerModal (BottomSheet)
19	    → getTeamDiaryStats(homeTeam)     ← jikgwan_records
20	    → computeTeamStreak(year, teamId) ← cachedAllDailyScores (시범경기 제외)
21	    → getJikgwanRecords() → computeStreakStats() → 내 연승
22	    → resolveHashtags(teamStreak, myStreak, gameResult, context)
23	    → StickerContent 렌더링 (ViewShot ref)
24	    → captureRef() → Clipboard.setImageAsync() → Sharing.shareAsync() fallback
25	```
26

81	BottomSheet 내부:
82	```
83	[헤더] "스티커 만들기" + 닫기 버튼

91	```
92	
93	데이터 로딩 (mount 시):
94	```
95	getTeamDiaryStats(homeTeam)  → stats (승률/승/무/패)
96	computeTeamStreak(year, homeTeam) → teamStreak
97	getJikgwanRecords() → computeStreakStats() → myStreak
98	→ resolveHashtags() → teamTag, myTag 자동 설정
99	```
100	
101	**handleCopy()**:
102	```typescript
103	try {
104	  const base64 = await captureRef(viewRef, { format: "png", result: "base64" });
105	  try {
106	    await Clipboard.setImageAsync(`data:image/png;base64,${base64}`);
107	    Alert.alert("완료", ...);
108	  } catch {
109	    // fallback
110	    const uri = await captureRef(viewRef, { format: "png", result: "tmpfile" });
111	    await Sharing.shareAsync(uri, { mimeType: "image/png" });
112	  }
113	} catch {
114	  Alert.alert("오류", "스티커 생성에 실패했습니다.");
115	}
116	```
