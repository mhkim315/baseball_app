15	## 배경
16	
17	모바일 앱(fullcount.kr)의 경기 상세 화면에서 사용자가 직관한 경기 결과를 인스타그램 스토리에 공유할 수 있는 동적 스티커를 생성하는 기능.
18	사용자가 직접 경기장에서 찍은 사진이나 티켓 없이도, 앱이 보유한 경기 데이터와 사용자의 직관 통계를 조합하여 인스타 스티커를 만들어 클립보드에 복사해준다.
19

22	## 데이터 흐름
23	
24	```
25	경기 상세 화면 → [스티커 만들기] 버튼
26	  → StickerEditorModal (bottom sheet)
27	    → 게임 데이터 조회: cachedDailyScores(date) | cachedGameDetail(gameId)
28	    → 사용자 통계 조회: computeDiaryStats(), computeStreakStats()
29	    → 팀 연승 계산: computeTeamStreak()
30	    → 해시태그 로직: resolveHashtags()
31	    → ViewShot.captureRef() → Clipboard.setImageAsync()
32	    → 실패 시: Sharing.shareAsync() fallback
33	```

52	### Phase 1: 데이터 레이어
53	
54	#### 1a. 팀 연승 계산 함수 — `lib/stats.ts` (신규 함수)
55	
56	```typescript
57	export function computeTeamStreak(games: ScoreEntry[], teamId: string): { type: "W" | "L" | "D" | null; count: number; display: string }
58	```
59	
60	- 입력: `cachedAllDailyScores(year)` 결과를 필터링하여 특정 팀 경기만 추출
61	- 날짜 내림차순 정렬, 시즌 단위 (시즌 넘어가면 초기화)
62	- 포스트시즌 포함, 시범경기 제외
63	- **※ ScoreEntry에 game_type 필드 없음** → 시범경기 날짜를 `EXHIBITION_SCORES`(별도 데이터소스)에서 추출하여 필터링. `cachedAllDailyScores()`가 머지한 결과에서 시범경기 날짜의 경기를 제외한 뒤 streak 계산.
64	- 결과: `{ type: "W", count: 5, display: "5연승" }` / `{ type: "W", count: 1, display: "연패탈출" }` 등
65	- **※ 엣지 케이스**: 무승부는 streak 유지 (끊지도 늘리지도 않음)
66	
67	#### 1b. 해시태그 규칙 엔진 — `lib/sticker.ts` (신규)
68	
69	```typescript
70	export interface HashtagResult {
71	  teamTag: string | null;   // 자동: 팀 streak 기반
72	  myTag: string | null;     // 자동: 내 직관 streak 기반
73	}
74	
75	export function resolveHashtags(
76	  teamStreak: { type: string; count: number },
77	  myStreak: { type: string; count: number },
78	  isFirstWin: boolean,
79	  isHome: boolean | null,
80	  gameResult: "win" | "lose" | "draw"
81	): HashtagResult
82	```
83	
84	| gameResult | 우선순위 | 조건 | 태그 |
85	|---|---|---|---|
86	| **win** | 1 | 2연패+ → 1승 | `#연패탈출` |
87	| | 2 | 2연승+ | `#N연승` |
88	| | 3 | 무승부 | `#무승부` |
89	| | - | 그 외 | 생략 |
90	| **my** | 1 | 해당팀 직관 첫 승 | `#직관첫승` |
91	| | 2 | 2연패+ → 1승 | `#직관연패탈출` |
92	| | 3 | 2연승+ | `#직관N연승` |
93	| | 4 | 1승 (홈) | `#홈승리` |
94	| | 5 | 1승 (원정) | `#원정승리` |
95	| | 6 | 첫 직관 경기 | `#첫직관` |
96	| | - | 그 외 | 생략 |
97	| **lose** | - | 자동 태그 전부 숨김 | 사용자 직접 입력만 |
98	| **draw** | 1 | 무승부 | `#무승부` |

212	#### 3a. ViewShot 캡처 + 클립보드
213	
214	```typescript
215	import ViewShot from "react-native-view-shot";
216	import * as Clipboard from "expo-clipboard";
217	import * as Sharing from "expo-sharing";
218	
219	async function handleCopy(viewRef: ViewShot) {
220	  let base64 = "";
221	  setCapturing(true);
222	  try {
223	    base64 = await viewRef.capture({ format: "png", result: "base64" });
224	  } catch (e) {
225	    Alert.alert("오류", "스티커 생성에 실패했습니다.");
226	    setCapturing(false);
227	    return;
228	  }
229	  try {
230	    await Clipboard.setImageAsync(`data:image/png;base64,${base64}`);
231	    Alert.alert("완료", "클립보드에 복사되었습니다.");
232	  } catch (e) {
233	    console.warn("Clipboard.setImageAsync failed", e);
234	    // fallback: 공유 시트
235	    try {
236	      const uri = await viewRef.capture({ format: "png", result: "tmpfile" });
237	      await Sharing.shareAsync(uri, { mimeType: "image/png" });
238	    } catch (e2) {
239	      Alert.alert("오류", "스티커 공유에 실패했습니다.");
240	    }
241	  } finally {
242	    setCapturing(false);
243	  }
244	}
245	```
246	
247	**변경사항**: `result: "tmpfile"` → `result: "base64"` (expo-clipboard `setImageAsync()`는 `data:image/png;base64,...` 포맷과의 호환성이 더 안정적)<br>
248	**uri 스코핑 수정**: `let base64`를 try 블록 밖에서 선언, 캡처 실패 시 early return
249	
250	> **참고**: `expo-clipboard`는 현재 미설치 상태 — `npx expo install expo-clipboard` 필요.

289	## 해시태그 편집 UX
290	
291	- 자동 생성된 해시태그는 음영/기본값 표시
292	- 사용자가 태그를 탭하면 `TextInput`으로 전환 (inline edit)
293	- `#` 기호는 고정 (수정 불가), 텍스트 부분만 편집 가능
294	- 3개 고정 슬롯: [팀태그] [내태그] [사용자태그]
295	- 사용자태그 플레이스홀더: "태그입력" (회색)
296	- 모달 닫으면 초기화 (저장하지 않음)
297	
298	실제 구현:
299	```tsx
300	// 각 태그 슬롯 구조
301	<View style={{ flexDirection: "row", alignItems: "center" }}>
302	  <Text style={styles.hashSign}>#</Text>
303	  {editing ? (
304	    <TextInput ... />
305	  ) : (
306	    <Pressable onPress={() => setEditing(index)}>
307	      <Text style={...}>{text || placeholder}</Text>
308	    </Pressable>
309	  )}
310	</View>
311	```
